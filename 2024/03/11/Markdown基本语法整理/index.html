Markdown基本语法整理

发表于2024-03-11|更新于2024-03-11
|阅读量:

标题

#号作为开头，后面跟内容
#和标题之间至少空一格
#的个数代表标题的级别，最多为6个
可以跨越层级，但请明确好关系

段落

要创建段落时，请请使用空白行将文本进行分隔。
不要用空格（spaces）或制表符（ tabs）缩进段落。

换行

在一行的末尾添加两个或多个空格，然后按回车键,即可创建一个换行。
也支持使用HTML的<br>标签进行换行
少数轻量级标记语言支持直接回车(enter)换行

字体

*斜体* __也是斜体__
**加粗** __也是加粗__
***粗斜体*** ___ 也是粗斜体___
`行内代码`
~~删除线~~
<u>下划线</u>

引用

要创建块引用，请在段落前添加>
块引用可以包含多个段落,段落之间的空白行也要添加一个>符号
块引用可以嵌套。在要嵌套的段落前添加>>

列表

有序列表

每个列表项前添加数字并紧跟一个英文句点，数字不必按数学顺序排列，但是列表应当以数字 1 起始

无序列表

破折号 (-)、星号 (*) 或加号 (+)然后接内容
同一层级的符号要相同
缩进一个或多个列表项可创建嵌套列表
如果一个项中要包含内容，需要换一行然后加一次缩进

分割线

使用 * - _ 中任意一个字符重复至少三次
为了兼容性，请在分隔线的前后均添加空白行

代码块

空行并缩进创建一个代码块

print("hello,world")

图片

格式:![图片名称](图片位置)
图片名可以省略
位置可以是链接，也可以是本地文件路径
图片不会嵌入md文件，提交时请附带上所有素材

链接

格式：[外面显示的内容](网页链接)
e.g 哔哩哔哩 —— [哔哩哔哩](www.bilibili.com)

表格

每个单元格的内容要用|分开

  • 内容中使用|要用\转义

第二行一定要有，用来规范整列的对齐方式

  • |–| 或 |：–| 左对齐
  • |–:| 右对齐
  • |:–:| 居中对齐
  • 的个数可以随意
